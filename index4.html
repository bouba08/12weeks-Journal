Javascript DOM

The Document Object Model (DOM) is allow us to easily access and manipulate tags, IDs, classes, Attributes, or Elements of HTML using commands or methods provided by the Document object. Using DOM, the JavaScript gets access to HTML as well as CSS of the web page and can also add behavior to the HTML elements. To reinforce my understanding of JavaScript DOM, we engaged in a practical exercise where we designed a dynamic web page featuring interactive buttons. Each button, when clicked, seamlessly replicated the resonating sound of a drum. This hands-on approach not only enhanced my proficiency in manipulating the Document Object Model but also provided a playful and engaging experience, making the learning process both enjoyable and insightful. Through this exercise, I gained practical insights into the seamless integration of JavaScript with HTML, paving the way for more interactive and user-friendly web applications.

Mongodb

After learning SQL we started doing mongodb, mongodb is a Nosql (not only sql), is an approach to database design that enables the storage and querying of data outside the traditional structures like SQL, found in relational databases. MongoDB is built on a scale-out architecture that has become popular with developers of all kinds for developing scalable applications with evolving data schemas. As a document database, MongoDB makes it easy for developers to store structured or unstructured data. It uses a JSON-like format to store documents. Differences between SQL and NoSQL are that SQL databases are used to store structured data while NoSQL databases like MongoDB are used to save unstructured data.
